InDecision

Android Studio • Java

Like many others, I have some trouble making perfectly arbitrary decisions. What kinda food should I buy? What game should I play? Music? There are so many options. InDecision is a response to this ever-present indecisiveness.

How it works

The user will have a moment of indecision, where perhaps they may be unable to decide which movie to see, or at which restaurant to eat.

Recognising this, the user would create a group in InDecision...

...and fill it with all their potential options.

Then, at the tap of a button, the user can receive a random suggestion.

InDecision will save the group and its contents so it can be accessed if issue ever comes up again.

Core Creeds

Simple

Users should be able to spontaneously create new groupings quickly and with minimal interaction.

Fast

Users should be able get a result instantly. Results should be obtainable without leaving the front page.

Reusable

Users should ideally use this app in their day to day moments of indecision. The previous two points influence this one.

Subtlety

Even the smallest interactions have been given subtle but satisfying responses. I've used apps before that lacked these small details, and while they were perfectly functional, the experience felt rigid. I aimed to prevent that wherever possible, without sacrificing InDecision's core of simplicity.

Going Forward

I'm going to keep up support for this app throughout beta testing and release. I'm looking to make the experience for users as pleasant as possible, so my first priority is bug squashing.

Once the majority of bugs are fixed and quality of life improvements have been made as per user suggestion, I'll transition into creating more features.

Some potential additions include the ability to favorite groups, sorting and filters, and better ways to rapidly add items to groups.
